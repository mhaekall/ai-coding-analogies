Memahami "Sistem" Web App

Dari HTML sederhana hingga Next.js, semua framework mengikuti prinsip yang sama: Analogi Restoran. Jika kamu paham alurnya, kamu bisa membuat perintah AI yang jauh lebih akurat.

Frontend / Client-Side

1. Ruang Makan (UI/UX)

Teknologi: HTML, CSS, React, Astro

Ini adalah tempat pelanggan duduk, melihat menu, dan menekan tombol pesanan. Semua yang kamu lihat di layar HP (warna, tombol, animasi) ada di sini.

Di Proyek "agecalc" (Sederhana)

Hanya punya Ruang Makan. Koki-nya langsung masak di sebelah meja pelanggan pakai script.js.

Di Proyek "Next.js" (Modern)

Ruang Makan diurus oleh React. Tugasnya cuma nampilin data secepat mungkin, nggak mikirin hitungan berat.

Backend / Server-Side / API

2. Dapur & Pelayan (Logic)

Teknologi: Node.js, Next.js API Routes, Python

Pelanggan tidak boleh masuk dapur. Pelayan (API) membawa pesanan dari Ruang Makan ke Dapur (Server). Di sinilah "otak" berat bekerja, seperti mengecek password, atau mengirim pesan ke ChatGPT.

Kenapa Butuh Dapur (Server)?

Agar rahasia aman. Kamu tidak mungkin menaruh API Key OpenAI di Ruang Makan (Frontend), karena pelanggan bisa mencurinya!

Keajaiban Next.js

Next.js spesial karena dia menggabungkan Ruang Makan + Dapur dalam satu gedung (satu repository). Ini yang disebut Fullstack.

Database / Storage

3. Gudang Bahan Makanan

Teknologi: Supabase, MongoDB, Firebase

Tempat menyimpan semua data secara permanen. Jika mati lampu (aplikasi direfresh), data di gudang tidak akan hilang.

Contoh Penggunaan

Menyimpan history obrolan chatbot, menyimpan data user yang mendaftar, atau menyimpan link untuk Link-in-bio.

Alur Kerja Utuh

Ruang Makan (UI) ➜ minta data ke Dapur (Server) ➜ Dapur ambil di Gudang (DB) ➜ Dapur masak ➜ Pelayan antar kembali ke Meja (UI).
